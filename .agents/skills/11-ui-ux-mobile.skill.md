# Instrucciones de la Skill

## Principio principal

La app debe diseñarse primero para celular.

Rango base:

```txt
360px - 430px de ancho
```

## Reglas visuales

- Usar tarjetas grandes.
- Botones fáciles de tocar.
- Formularios cortos.
- Nada de tablas complejas en móvil.
- Priorizar lectura rápida.
- Evitar pantallas saturadas.
- Usar navegación inferior.
- Mostrar feedback inmediato después de guardar.

## Navegación principal

Usar navegación inferior tipo app móvil:

```txt
Inicio
Entrenar
Comidas
Progreso
Ajustes
```

## Accesibilidad táctil

- Botones mínimo 44px de alto.
- Espaciado cómodo entre acciones.
- Inputs fáciles de tocar.
- No poner acciones destructivas demasiado cerca de acciones principales.

## Dashboard

Debe mostrar:

- Saludo.
- Resumen del día.
- Último check-in.
- Entrenamientos de la semana.
- Último consejo.
- Accesos rápidos.

## Formularios

- Dividir por secciones si crecen mucho.
- Usar sliders o steppers para escalas 1–10.
- No pedir demasiados campos obligatorios.
- Permitir notas opcionales.

## Estilo visual

- Cards con bordes suaves.
- Buen contraste.
- Espaciado generoso.
- Íconos solo cuando ayuden.
- No saturar con colores.

## Tono visual

La app debe sentirse:

- Energética.
- Clara.
- Moderna.
- Cercana.
- Práctica.
